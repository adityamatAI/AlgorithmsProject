Complexity Analysis: O(V + E)

The time complexity of Breadth-First Search is exactly proportional to the number of Vertices (V) and Edges (E) in the graph: O(V + E).

  • O(V): Every vertex is enqueued and dequeued exactly once.
  • O(E): Every edge is examined exactly twice in an undirected graph (once from each of its endpoints).

Compared to DFS: Depth-First Search shares the exact same worst-case time complexity, O(V + E), but explores in a different order (going deep instead of wide).

Compared to Dijkstra's: Dijkstra's Algorithm handles weighted edges by using a Priority Queue, bringing its complexity to O((V + E) log V). BFS is much faster but only works for unweighted graphs.

Searching: Systematic Level-by-Level

BFS employs a systematic level-by-level searching mechanism. It fully explores all nodes at the present "depth" (distance from the start) before moving on to nodes at the next depth level.

Key Concept: The FIFO Queue Requirement To achieve this perfect level-by-level traversal, BFS relies strictly on a First-In-First-Out (FIFO) Queue. When a node is discovered, it is placed at the back of the queue. Because it always pulls from the front, it guarantees that all nodes discovered at distance d are fully processed before the algorithm even looks at nodes at distance d+1.

Queue (BFS) vs. Stack (DFS):

  • BFS (Queue): Oldest discovered nodes are processed first. This guarantees we check distance 1 nodes, then distance 2 nodes, spreading out evenly like a ripple in water.
  • DFS (Stack): Newest discovered nodes are processed first (LIFO). This causes the algorithm to plunge as deep as possible down a single path before eventually backtracking.

Paradigm: Greedy-Like Shortest Paths

BFS is effectively a Greedy-like strategy for unweighted graphs. At every step, it makes the locally optimal choice by exploring the absolute nearest unvisited nodes first.

Because it expands evenly, the moment BFS discovers a target node, it is guaranteed to have found the shortest path to that node (in terms of fewest edges). Look closely at the visualization above: as nodes turn queued/gray, a distance label (d=...) is stamped next to them. This distance is permanently optimal.

Key Limitation: Why BFS Fails on Weighted Graphs BFS assumes every edge has an identical "weight" or cost of 1. Therefore, the path with the fewest edges is always the shortest path. However, in weighted graphs (e.g., city distances), a path with 5 short edges might be much cheaper than a path with 1 massive edge. Because BFS doesn't look at weights, it will greedily choose the 1-edge path, returning an incorrect "shortest path". For weighted graphs, Dynamic Programming (Bellman-Ford) or greedy Priority Queues (Dijkstra's) are required.

Step-by-Step: BFS on a Tree

Let's trace the algorithm using the Tree preset. The tree edges are: A-B, A-C, B-D, B-E, C-F, C-G. We begin our search starting at the root node, A.

Step 1: Initialization

  • Start at A. Mark A as discovered (distance = 0) and add it to the queue.
  • Current Queue: [A]

Step 2: Process Level 0 (Node A)

  • Dequeue A. Explore its neighbors: B and C.
  • Mark B and C as discovered (distance = 1), and enqueue them. Mark A as fully explored.
  • Current Queue: [B, C]

Step 3: Process Level 1 (Node B)

  • Dequeue B. Explore its neighbors: D and E.
  • Mark D and E as discovered (distance = 2), and enqueue them. Mark B as fully explored.
  • Current Queue: [C, D, E]

Step 4: Process Level 1 (Node C)

  • Dequeue C. Explore its neighbors: F and G.
  • Mark F and G as discovered (distance = 2), and enqueue them. Mark C as fully explored.
  • Current Queue: [D, E, F, G]

Step 5+: Process Level 2 (Leaf Nodes)

  • Dequeue D, E, F, and G one by one. None of them have any undiscovered neighbors.
  • Each node is simply marked as fully explored.
  • Final Queue: [] (Empty)

Once the queue is completely empty, the while loop terminates and the BFS algorithm is complete!
