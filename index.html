Lecture & Theory: Mastering BFS

Algorithm Overview

Breadth-First Search (BFS) is a fundamental graph traversal algorithm that explores vertices "layer by layer". It starts at a source vertex and explores all of its immediate neighbors first, before moving on to the next level of neighbors.

How It Works

The algorithm uses a First-In-First-Out (FIFO) queue to manage which vertex to visit next. It systematically colors vertices to keep track of its progress:

  • White: The vertex is undiscovered.
  • Gray: The vertex has been discovered and is in the queue, waiting to be processed.
  • Black: The vertex and all of its neighbors have been fully explored.

When to Use BFS

BFS is the ideal algorithm for finding the shortest path (in terms of number of edges) between two vertices in an unweighted graph. Common applications include:

  • Finding the shortest route in a GPS or mapping system (where all roads have equal "cost").
  • Discovering all nearby friends in a social network ("friends of friends").
  • Web crawlers discovering all pages reachable from a starting URL.

Complexity Analysis: O(V + E)

The time complexity of Breadth-First Search is directly proportional to the number of Vertices (V) and Edges (E) in the graph: O(V + E).

  • Worst & Average Case Time Complexity: O(V + E). Every vertex is enqueued once, and every edge is examined.
  • Best Case Time Complexity: O(1) or O(V). O(1) if searching for a specific target node that happens to be the starting node. If traversing the entire graph, it remains O(V + E).
  • Space Complexity: O(V). The FIFO queue may need to store up to V vertices in the worst-case (e.g., a "star" graph).
Worst-Case Scenario: Dense Graphs A dense graph (available via the Worst-Case preset) is the worst-case scenario because the number of edges E approaches V². This forces the algorithm to check a massive number of edges even if the number of vertices V is small, maximizing the total operations.

Compared to Dijkstra's: Dijkstra's Algorithm handles weighted edges by using a Priority Queue, bringing its complexity to O((V + E) log V). For unweighted graphs, BFS is significantly faster.

Searching: Systematic Level-by-Level

BFS employs a systematic level-by-level searching mechanism. It fully explores all nodes at the present "depth" (distance from the start) before moving on to nodes at the next depth level.

Key Concept: The FIFO Queue Requirement To achieve this perfect level-by-level traversal, BFS relies strictly on a First-In-First-Out (FIFO) Queue. When a node is discovered, it is placed at the back of the queue. Because it always pulls from the front, it guarantees that all nodes discovered at distance d are fully processed before the algorithm even looks at nodes at distance d+1.

Impact of Changing the Searching Mechanism: If we replace the Queue with a Last-In-First-Out (LIFO) Stack, the algorithm becomes Depth-First Search (DFS). The critical impact of this change is that we completely lose the shortest-path guarantee. A Stack-based search will plunge deep down a single branch, potentially taking a long path to a node that was right next to the start.

Paradigm: Greedy-Like Shortest Paths

BFS is effectively a Greedy-like strategy for unweighted graphs. At every step, it makes the locally optimal choice by exploring the absolute nearest (one edge away) unvisited nodes first.

Because it expands evenly, the moment BFS discovers a target node, it is guaranteed to have found the shortest path to that node. Notice in the visualization how the distance label `d=...` is set once and is permanently optimal.

Counterexample: BFS vs DP (Bellman-Ford) on Weighted Graphs Consider a graph A-C (cost 10), A-B (cost 1), and B-C (cost 1). If we use BFS to find the shortest path from A to C, it greedily sees the direct path A->C is only 1 edge and declares it the shortest. This is wrong (cost 10). A Dynamic Programming algorithm (Bellman-Ford) would correctly find the path A->B->C (cost 2).

Step-by-Step: BFS on a Tree

Let's trace the algorithm using the Tree preset, starting at the root node, A.

Step 1: Init → Start at A. Mark it gray (distance = 0) and add to queue. Queue: [A]

Step 2: Dequeue A → Explore its neighbors: B and C. Mark them gray (distance = 1) and enqueue. Mark A black. Queue: [B, C]

Step 3: Dequeue B → Explore its neighbors: D and E. Mark them gray (distance = 2) and enqueue. Mark B black. Queue: [C, D, E]

Step 4: Dequeue C → Explore its neighbors: F and G. Mark them gray (distance = 2) and enqueue. Mark C black. Queue: [D, E, F, G]

Step 5+: Dequeue Leaves → Dequeue D, E, F, and G one by one. None have undiscovered neighbors. Mark each black. The queue empties.

Once the queue is empty, the algorithm terminates, having found all shortest paths from A.
